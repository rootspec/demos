# Level 3: Interaction Architecture

**References:** L1 (Philosophy), L2 (Truths)

---

## Core Interaction Model

RootFeed is a **browse-first, interact-second** product. Most visitors will scroll and read. Interactions (liking, following, composing) are available but not required. Every interactive element must respond without any perceptible delay — this supports the Grounded Immediacy pillar.

All state is client-side and ephemeral. Interactions that modify counts or lists do so in memory only.

---

## User Journeys

### Journey 1: The Feed Browser

**Trigger:** User arrives at `/`
**Loop:**
1. See the meta banner — understand this is a RootSpec demo
2. Scroll the home feed — read posts chronologically
3. Click a post to see its thread detail at `/post/[id]`
4. Click an author name to visit their profile at `/profile/[handle]`
5. Return to feed; optionally load more posts (10 at a time)
6. Write a new post via the composer — post appears at top of feed
7. Like or bookmark posts — counts update immediately

**Feedback signals:**
- Liked posts show filled heart icon; count increments by one
- Bookmarked posts show filled bookmark icon
- Composed posts appear at top of feed instantly with current timestamp

---

### Journey 2: The Profile Visitor

**Trigger:** User clicks a handle or navigates to `/profile/[handle]`
**Loop:**
1. See user's display name, handle, bio, follower/following counts
2. See follow/unfollow button (default: unfollow)
3. Click Follow — button changes to "Unfollow"; follower count increments by one
4. Click Unfollow — reverts to "Follow"; follower count decrements by one
5. Scroll user's posts
6. Click any post to go to thread detail

**Feedback signals:**
- Follow button state is visually distinct (filled vs. outline)
- Follower count updates immediately in the UI

---

### Journey 3: The Thread Reader

**Trigger:** User clicks a post or navigates to `/post/[id]`
**Loop:**
1. See the focal post with full content, author, engagement counts
2. If this post is a reply, see the parent post above it
3. See all replies below the focal post
4. Like the focal post — count updates
5. Click a reply author to visit their profile

**Feedback signals:**
- Focal post is visually emphasized (larger font or border treatment)
- Parent post is visually de-emphasized (muted, context-framing)
- Replies are visually indented or separated

---

### Journey 4: The Searcher

**Trigger:** User navigates to `/search`
**Loop:**
1. See empty state with search input focused
2. Type a keyword — results appear immediately as characters are entered (no submit button needed)
3. Results show matching posts with author links
4. Click a result to go to the post detail
5. Clear input — results disappear, empty state returns
6. If no results for current query, see a "No results" message

**Feedback signals:**
- Results update synchronously with each keystroke
- "No results" state shown only when query is non-empty and results are empty
- Query text highlighted in results (optional enhancement)

---

### Journey 5: The Explorer

**Trigger:** User navigates to `/explore`
**Loop:**
1. See trending tags (ordered by post count)
2. Click a tag — post list filters to that tag only; active tag highlighted
3. Click same tag again or click another tag — filter changes
4. See suggested users with bio, follower counts, follow button
5. See popular posts (ordered by engagement)
6. Click any post or profile to navigate

**Feedback signals:**
- Active tag is visually highlighted (filled vs. outline chip)
- Post list updates immediately when tag changes
- Follow state persists across the page session (follows from explore visible on profile)

## Global Interactions

### Dark/Light Theme Toggle

**Location:** Navigation bar (persistent)
**Behavior:**
- On first load: detect system preference (`prefers-color-scheme`)
- Manual toggle: switches between dark and light; stores preference in `localStorage`
- Preference persists across page navigation within session
- On page reload: restores stored preference, falls back to system preference

### Meta Banner

**Location:** Top of every page, above the nav
**Content:**
- Short description of the scaffold demo
- Link to scaffold commit on GitHub (absolute URL)
- Link to spec files on GitHub (absolute URL)
- Link to SEED.md on GitHub (absolute URL)

**Behavior:** Static — no interactions. Always visible. Cannot be dismissed.

### Post Composer

**Location:** Top of the home feed (`/`)
**Fields:** Text area for post content, submit button
**Behavior:**
- Submit button disabled when textarea is empty
- On submit: new post object created with current timestamp and placeholder author (first user in mock data)
- New post appears at top of the feed list immediately
- Textarea clears after submission
- Post content limited to [max_post_length] characters

### Load More

**Location:** Bottom of home feed
**Behavior:**
- Initial load: show first 10 posts
- Click "Load More": append next 10 posts to list
- When all posts displayed: button hidden or shows "All caught up"

## Error and Empty States

| Context | Trigger | Display |
|---|---|---|
| Search | Query entered, no matches | "No results for [query]" message |
| Feed | All posts loaded | "All caught up" text, Load More hidden |
| Explore tags | Tag selected with no posts | "No posts tagged #[tag]" message |

No 404 handling is specified — only known routes are built.

## Navigation

The persistent nav includes: **RootFeed** (home link), **Home**, **Explore**, **Search**.
Active route is highlighted.
Profile pages are accessed via post/explore links — no direct nav item for profiles.
