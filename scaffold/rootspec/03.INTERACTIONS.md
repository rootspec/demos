# Level 3: Interaction Architecture

## Interaction Model

RootFeed has two audiences experiencing the same app differently:

- **Evaluator** — A developer assessing RootSpec. They interact with the social feed but their real journey is understanding *how* it was built.
- **Explorer** — Someone who landed on the demo and is just clicking around. They experience the feed as a feed first, and notice the meta layer second.

Both paths converge: the feed is engaging enough to explore, and the meta layer is visible enough to discover without searching for it.

## Core Interaction Loops

### Immediate Scale — Single Actions (seconds)
| Trigger | Action | Feedback |
|---------|--------|----------|
| See a post | Read content, scan engagement counts | Content renders with author context |
| Tap like | Toggle like state | Count updates, icon state changes instantly |
| Tap bookmark | Toggle bookmark state | Icon state changes instantly |
| Tap follow | Toggle follow state on profile | Button state and follower count update |
| Type in composer | Enter post content | Character input reflected in real time |
| Submit post | Publish to feed | Post appears at top of feed immediately |
| Type in search | Enter keywords | Results filter as user types |

### Session Scale — Connected Actions (minutes)
| Journey | Flow | Outcome |
|---------|------|---------|
| Feed browsing | Scroll → read → like → scroll → load more | Developer sees a living feed with realistic content |
| Profile exploration | Tap author → view profile → see their posts → follow | Developer sees interconnected data |
| Thread reading | Tap post → see parent/replies → navigate thread | Developer sees threading works |
| Discovery | Visit explore → browse trending tags → filter by tag → find posts | Developer sees search and filtering work |
| Post creation | Open composer → write → submit → see it in feed | Developer sees client-side state management |

### Extended Scale — Cross-Session (the demo visit)
| Phase | Experience | Supports Pillar |
|-------|-----------|-----------------|
| Landing | Feed loads, meta banner visible, content feels real | Effortless Curiosity |
| Exploration | Click around, interactions work, everything feels connected | Effortless Curiosity |
| Meta discovery | Notice banner, click through to spec/seed/scaffold commit | Earned Credibility |
| Verification | See spec structure, see tests, understand the pipeline | Earned Credibility |
| Decision | "I should try this" — leaves with intent to use RootSpec | Approachable Confidence |

## Screen Architecture

### Home Feed (`/`)
- Chronological post timeline with author avatars, handles, content, timestamps, engagement counts
- Like and bookmark toggles on each post
- Post composer at the top
- Load more pagination
- Meta banner (persistent across all pages)

### Profile (`/profile/[handle]`)
- User bio, avatar, follower/following counts
- Follow/unfollow toggle
- User's posts listed below profile header

### Post Detail (`/post/[id]`)
- Full post with engagement actions
- Parent post displayed above if this is a reply
- Replies displayed below as a thread
- Like and bookmark toggles

### Search (`/search`)
- Text input with live filtering
- Tag filter chips
- Results update as user types

### Explore (`/explore`)
- Trending tags with post counts
- Suggested users to follow
- Popular posts by engagement

### Meta Banner (persistent)
- Explains the demo's origin — seed file → spec → implementation
- Links to SEED.md, spec files, scaffold commit
- Always visible, never dismissable

## Navigation Pattern

Global navigation provides access to all five routes. Current route is visually indicated. All post and profile references are clickable, creating a web of interconnected content.

## Theme

Dark and light modes via system preference detection with manual toggle. Theme persists in client-side storage within a session.

## Failure Modes

| Scenario | Behavior |
|----------|----------|
| Empty search results | "No posts found" message with suggestion to try different keywords |
| Missing user profile | Graceful fallback — "User not found" |
| Invalid post ID | Redirect to home feed or "Post not found" |
| Composer with empty/whitespace content | Validation prevents submission |
| Page refresh | Client-side state resets (likes, follows, new posts) — acceptable per L2 constraints |

## Verification Criteria

- Every screen listed above has at least one user story with Cypress tests
- Every interaction in the immediate scale table produces observable feedback
- Navigation between all routes works without errors
- Theme toggle works and respects system preference
- Meta banner is visible on every page
