## Interaction Loop Architecture

| Scale     | Scope                        | Duration    | Key Patterns                        |
|-----------|------------------------------|-------------|-------------------------------------|
| Immediate | Single action                | Instant     | Like toggle, follow toggle, search  |
| Session   | Navigating the app           | Minutes     | Feed browsing, thread reading       |
| Extended  | Exploring the demo           | Several min | Reading spec links, scaffold origin |
| Lifetime  | Developer's product decision | Once        | Deciding whether to adopt RootSpec  |

---

## Core Interaction Patterns

### Pattern 1: Feed Browsing

The home feed is the primary surface. The user scrolls a chronological list of posts. Each post shows author avatar, display name, handle, content, timestamp, like count, and repost count.

**Trigger:** User visits `/`
**Loop:**
1. Feed renders first [N] posts
2. User scrolls down
3. "Load more" button appears at the bottom
4. User clicks → next [N] posts append to list
5. Repeat until all posts are shown

**Feedback:** Post count updates in the feed; no spinner — content appends instantly (static data, no latency).

---

### Pattern 2: Post Engagement (Like / Bookmark)

Each post in the feed and on the post detail page has like and bookmark toggles.

**Trigger:** User clicks the like or bookmark icon on a post
**Loop:**
1. Icon changes state (filled vs. outline)
2. Like count increments or decrements
3. State is local to the session — reloading resets to default

**Feedback:** Immediate visual toggle with count update. No loading state.

**Constraint (from L2):** State is not persisted. Reloading the page resets all toggles.

---

### Pattern 3: Follow / Unfollow

User profile pages show a follow/unfollow button. The follow state is client-side only.

**Trigger:** User clicks "Follow" or "Unfollow" on a profile page
**Loop:**
1. Button text toggles between "Follow" and "Unfollow"
2. Follower count increments or decrements by one
3. State is local to the session

**Feedback:** Button label and count update immediately. No loading state.

---

### Pattern 4: Search

The search page filters posts by keyword as the user types.

**Trigger:** User types in the search input on `/search`
**Loop:**
1. Results filter reactively as query changes
2. Posts matching query (case-insensitive substring) are shown
3. Empty state shown when query is non-empty but no matches exist
4. No results shown when query is empty

**Feedback:** Immediate reactive filtering with no debounce (static data, no latency concern).

---

### Pattern 5: Tag Filtering (Explore)

On the explore page, tags are clickable. Clicking a tag filters the post list to that tag.

**Trigger:** User clicks a tag on `/explore`
**Loop:**
1. Tag becomes visually selected
2. Post list below filters to posts with that tag
3. Clicking the same tag again (or a clear button) deselects and shows all posts
4. Clicking a different tag switches the filter

**Feedback:** Immediate reactive filter. Selected tag has distinct styling.

---

### Pattern 6: Post Composer

A post composer is accessible from the home feed. The user can write and submit a new post that appears in the feed for the current session.

**Trigger:** User clicks "New Post" or a compose button on `/`
**Loop:**
1. Composer input expands (or a modal/panel appears)
2. User types content (up to [N] characters)
3. Character count shown
4. User submits
5. New post appears at the top of the feed with a placeholder author identity

**Feedback:** Post appears immediately at top of feed. Composer resets. No server round-trip.

**Constraint (from L2):** Composed posts do not persist across page loads.

---

### Pattern 7: Theme Toggle (Dark / Light)

The site supports dark and light themes. System preference is detected on load. User can override manually.

**Trigger:** User clicks the theme toggle button (visible in navigation)
**Loop:**
1. Theme switches between light and dark
2. Preference is stored in localStorage for the session
3. On next load, stored preference is applied; system preference used if no stored value

**Feedback:** Immediate full-page theme switch. Toggle icon reflects current theme.

---

### Pattern 8: Meta Banner Navigation

The persistent meta banner explains the RootSpec origin and links to external resources.

**Trigger:** Page load (banner is always visible)
**Affordances:**
- "View the scaffold commit →" — links to GitHub commit
- "View the spec →" — links to spec files in repo
- "View the seed →" — links to SEED.md in repo

**Feedback:** Standard link navigation. No interactive state.

## Navigation Model

Navigation is link-based (SvelteKit routing). No back-stack management beyond browser defaults.

| Route | Purpose |
|-------|---------|
| `/` | Home feed — primary surface |
| `/explore` | Trending tags, suggested users, popular posts |
| `/search` | Keyword search across posts |
| `/profile/[handle]` | User profile with their posts |
| `/post/[id]` | Single post with thread (parent + replies) |

Navigation appears in the top nav bar on every page. Profile and post links are embedded in post cards and profile listings.

## Failure Modes and Edge Cases

| Scenario | Behavior |
|----------|----------|
| Search with no results | Empty state message: "No results for '[query]'" |
| Explore with no active tag filter | All posts shown in post list |
| Post thread with no replies | Replies section is hidden |
| Post thread with a parent post | Parent post shown above the main post |
| User navigates to pre-rendered route | Page renders normally |
| User navigates to unknown route (not in mock data) | Not specced — static site has no such pages |

---

## Verification Criteria

- Feed renders all mock posts with correct author attribution
- Like toggle increments/decrements count immediately
- Follow toggle updates button state and follower count immediately
- Search filters reactively as user types
- Tag filter on explore shows only matching posts
- Composer creates a post visible at the top of the feed
- Theme toggle switches between dark and light modes
- Meta banner is visible on every page with all three links functional
- Navigation links are correct on all five routes
