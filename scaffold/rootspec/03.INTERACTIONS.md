# Level 3: Interaction Architecture

**References:** 01.PHILOSOPHY.md, 02.TRUTHS.md

## Core User Journey (First 5 Minutes)

1. Developer lands on the deployed URL
2. They see the meta banner immediately — "RootFeed started as a bare SvelteKit scaffold..." — with links to seed, spec, and scaffold commit
3. They see the home feed: a timeline of posts with avatars, handles, timestamps, like/repost counts
4. They click a post → post detail page with the full thread (parent and replies)
5. They click an author handle → profile page with bio, follower/following counts, follow/unfollow button
6. They use the composer to write and publish a post — it appears at the top of the feed
7. They navigate to Search, type a keyword — results update in real time
8. They navigate to Explore — see trending tags, click a tag to filter posts
9. They toggle the theme button in the nav — switches between light and dark
10. They click "View the spec" in the meta banner — opens GitHub to the rootspec/ directory

## Primary Interaction Loops

### Feed Loop (daily-use equivalent)
**Trigger:** User arrives at `/`
**Flow:** Scroll feed → like/bookmark posts → click into post detail → read thread → click author → view profile → follow/unfollow → return to feed
**Feedback:** Like button toggles red fill + count increments. Bookmark button toggles amber fill. Follow button toggles between "Follow" and "Following" labels.

### Discovery Loop
**Trigger:** User navigates to `/explore`
**Flow:** Scan trending tags → click tag → posts filtered by that tag → click into post or author
**Feedback:** Selected tag is visually highlighted. Filtered posts appear below the tag row.

### Search Loop
**Trigger:** User navigates to `/search`
**Flow:** Type in search box → results update as user types (no submit required) → click result to go to post
**Feedback:** Results appear immediately below the input. Empty state shown when query has no matches.

### Compose Loop
**Trigger:** User focuses the composer textarea on `/`
**Flow:** Type content (up to [post_max_chars] chars) → character count updates live → press "Post" → post appears at top of feed
**Feedback:** Character counter (e.g., `0/300`) updates on every keystroke. Error message shown if user tries to submit empty post. After submission, textarea clears and new post is visible in feed.

### Thread Loop
**Trigger:** User clicks a post anywhere in the app
**Flow:** Post detail shows full content + timestamp. If post is a reply, parent post is shown above. All replies are listed below.
**Feedback:** Replies are visually indented / offset from the main post.

### Profile Loop
**Trigger:** User clicks an author name or handle anywhere
**Flow:** Profile shows display name, handle, bio, follower/following counts, follow/unfollow button, and the user's posts
**Feedback:** Follow button label switches immediately on click (client-side only). Follower count does NOT change (no server — client-side only state).

## Interaction Patterns

### Toggle (Like, Bookmark, Follow)
- Active state: colored icon (red for like, amber for bookmark, blue label "Following" for follow)
- Inactive state: outline icon (gray/muted)
- No animation required — visual feedback via icon fill and color change is sufficient
- Count displayed next to like/repost shows live-updated value based on toggle state

### Real-Time Filter (Search, Tag Filter)
- No submit button required — results update on every keystroke or tag click
- Empty query on Search page shows no results (not an error)
- Tag click on Explore page filters the posts section inline

### Post Composer
- Located at top of home feed, above the post list
- Character counter always visible
- Post button disabled-looking when content is empty
- On successful submission: textarea clears, new post appears at top of feed with correct author (always alice.dev)
- On invalid submission (empty): inline error appears below textarea

### Theme Toggle
- Located in the navigation bar
- Clicking toggles between light and dark
- Preference persisted in localStorage under key `rootfeed-theme`
- On first visit: system preference used (via `prefers-color-scheme`)
- All surfaces update immediately on toggle (no page reload)

## Failure Modes and Edge Cases

| Situation | Behavior |
|---|---|
| Post not found (`/post/[id]` with invalid ID) | Show "Post not found." message — no error page, no redirect |
| User not found (`/profile/[handle]` with invalid handle) | Show "User not found." message |
| Search with no results | Show "No results for [query]" inline — not an error |
| Composer submitted empty | Show inline validation error "Post cannot be empty" |
| Composer post content exceeds [post_max_chars] chars | Character counter shows overage; submit blocked |
| Navigating directly to an internal route | SvelteKit static routing handles it; no 404 for valid routes |

## Navigation Model

The persistent nav (always visible) contains:
- RootFeed logo/name (links to `/`)
- Home link
- Explore link
- Search link
- RootSpec version badge (v7.2.5)
- Theme toggle button

Profile pages are accessed by clicking author names. Post detail accessed by clicking post content. There is no back button in the nav — browser back is the expected interaction.

## Meta Banner

Always visible at the top of every page, above the main content, below the nav. Contains:
- Explanatory text about the scaffold story
- Link: "View the seed" → SEED.md on GitHub
- Link: "View the spec" → rootspec/ directory on GitHub
- Link: "View the scaffold commit" → specific commit hash on GitHub

The banner is never dismissable. It is a permanent fixture of the demo framing.

## Load More (Feed Pagination)

The home feed loads [initial_post_count] posts initially. A "Load more" button appears below the initial set. Clicking it appends the next [page_size] posts to the feed. User-composed posts always appear at the top of the feed regardless of pagination state.
