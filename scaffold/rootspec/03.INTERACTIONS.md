## Core User Journeys

### Journey 1: First-Time Developer Visit
1. User lands on `/` (Home Feed) — sees persistent meta-banner at top explaining this is a RootSpec scaffold demo
2. Scans the feed — sees real-ish posts with avatars, handles, timestamps, like/repost counts
3. Clicks a post → navigates to `/post/[id]` — sees full thread (parent post if reply, all replies)
4. Clicks an author name → navigates to `/profile/[handle]` — sees bio, follower/following counts, posts
5. Returns to feed via nav — clicks Explore, searches for a tag, follows a user

### Journey 2: Interactive Exploration
1. On any post: clicks Like → count increments, button toggles state
2. On any post: clicks Bookmark → icon toggles state
3. On `/profile/[handle]`: clicks Follow → button changes to Unfollow
4. On `/`: clicks "Load more" → next 10 posts append to feed
5. On `/`: opens post composer → types a post → submits → new post appears at top of feed
6. On `/explore`: clicks a tag → feed filters to posts with that tag
7. In nav: clicks theme toggle → UI switches between dark/light mode

### Journey 3: Search
1. User navigates to `/search`
2. Types in search box — results update live as they type (no submit required)
3. Results show matching posts with author name linked to profile
4. Clearing the query shows empty state (no results shown until user types)

## Interaction Patterns

### Feed Pagination
- **Trigger:** User clicks "Load more" button below the feed
- **Behavior:** Next 10 posts append to existing list; button hides when no more posts remain
- **Initial state:** First 10 posts visible on load
- **Feedback:** Posts appear immediately (no loading state required for mock data)

### Like Toggle
- **Trigger:** Click the like button on any post
- **Behavior:** Like count increments (+1) and button enters active state; clicking again decrements (-1) and returns to inactive
- **Scope:** Client-side only; resets on page reload
- **Feedback:** Immediate visual state change

### Bookmark Toggle
- **Trigger:** Click the bookmark button on any post
- **Behavior:** Bookmark icon toggles between unfilled (inactive) and filled (active) state
- **Scope:** Client-side only; resets on page reload
- **Feedback:** Immediate icon change

### Follow/Unfollow
- **Trigger:** Click Follow button on profile page
- **Behavior:** Button label changes to "Unfollow"; clicking again returns to "Follow"
- **Scope:** Client-side only; does not affect follower count in this implementation
- **Feedback:** Immediate button state change

### Post Composer
- **Trigger:** User clicks "New Post" or composer area on home feed
- **Behavior:** Text area appears; user types content; submits via button; new post appears at top of feed with current timestamp and a placeholder author identity
- **Scope:** Client-side only; post disappears on reload
- **Feedback:** Post appears immediately at top of feed after submission; composer resets

### Tag Filter (Explore)
- **Trigger:** Click a tag on the Explore page
- **Behavior:** Posts section below filters to show only posts tagged with the selected tag; clicking the same tag again deselects and shows all posts
- **Feedback:** Post list updates immediately; selected tag is visually highlighted

### Theme Toggle
- **Trigger:** Click theme toggle button in nav/header
- **Initial state:** Detected from system preference (prefers-color-scheme)
- **Behavior:** Switches between dark and light mode; preference persisted in localStorage
- **Feedback:** Entire UI transitions; toggle button reflects current mode

### Real-Time Search
- **Trigger:** User types in search input on `/search`
- **Behavior:** Results filter on every keystroke; case-insensitive substring match against post content
- **Empty state:** No results shown when query is empty; "No results" message when query has no matches
- **Feedback:** Results update instantly as user types

## Navigation Structure

```
/ (Home Feed)
  ├── /post/[id] (Post Detail + Thread)
  ├── /profile/[handle] (User Profile)
  ├── /explore (Trending Tags + People)
  └── /search (Live Search)
```

- Nav bar always visible with links: Home, Explore, Search
- RootFeed wordmark/logo in nav links to Home
- RootSpec version (v7.2.7) visible in nav or header
- Meta-banner appears at top of every page (persistent, not dismissible)
- Footer appears at bottom of every page

---

## Failure Modes and Edge Cases

- **No search results:** Show "No results for '[query]'" message
- **No posts for tag filter:** Show "No posts tagged #[tag]" message
- **Empty composer submission:** Ignore or show inline validation; do not create empty posts
- **Feed exhausted:** Hide "Load more" button when all posts are loaded
- **Unknown routes:** Static build produces no page for unknown dynamic parameters — this is acceptable per TRUTHS.md
