# RootSpec Demos

Demonstrates the RootSpec spec-to-code pipeline across three project scenarios. Each demo follows the full workflow — from product idea to deployed, validated application.

## Demos

| Demo | Scenario | Status |
|------|----------|--------|
| [greenfield/](greenfield/) | Marketing site for RootSpec, built from scratch | In progress |
| scaffold/ | Add RootSpec to an existing scaffold | Planned |
| brownfield/ | Retrofit RootSpec onto an existing codebase | Planned |

## How It Works

Each demo runs the four RootSpec skills in order:

1. `/rs-init` — Set up RootSpec directory structure
2. `/rs-spec` — Define the specification through interview
3. `/rs-impl` — Implement from the spec, test-driven
4. `/rs-validate` — Run tests and prove it works

The resulting apps deploy to GitHub Pages via CI.
