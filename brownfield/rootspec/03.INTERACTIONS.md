# Level 3: Interaction Architecture

**Product:** RootWeather
**Version:** 7.3.7
**Status:** Baseline

---

## Core Interaction Loops

### Loop 1: Search and Discover

**Trigger:** User arrives with a city in mind, or wants to check a new location.

1. User types in the search bar (minimum 2 characters)
2. After 300ms debounce, app queries Open-Meteo geocoding API
3. Dropdown shows up to 5 matching cities with region and country
4. User selects a city — view switches to weather, search field shows city name
5. App fetches weather data and renders current conditions, alerts, hourly, and 7-day forecast
6. User optionally saves the city to favorites via "⭐ Save Location" button

**Feedback signals:**
- Loading state while weather fetches
- Error state if fetch fails
- Empty state if no city is selected yet

---

### Loop 2: Favorites — Fast Switching

**Trigger:** User has saved locations and wants to switch between them quickly.

1. Favorites bar shows saved cities as pill-shaped chips (visible only when favorites exist)
2. Active city chip is highlighted (purple background)
3. User clicks a chip to switch — view stays on weather, new city's data loads
4. User can remove a city from favorites by clicking the ✕ on the chip
5. If removed city was the default city, default is cleared

**Feedback signals:**
- Active chip highlighted in brand purple
- Remove button visible on each chip with dimmed opacity, full on hover

---

### Loop 3: Dashboard — At-a-Glance Overview

**Trigger:** User wants to see all saved locations at once.

1. User clicks "Dashboard" in the view toggle (visible only when favorites exist)
2. App loads current weather for all favorites in parallel
3. Grid of location cards shows: emoji + temperature + humidity + wind for each city
4. Clicking a card navigates to that city's full weather view
5. If 2+ favorites exist, a "Compare" button appears in the dashboard header

**Feedback signals:**
- Loading state while all locations fetch
- "Unable to load" shown per-card on failure
- Cards highlight on hover with a subtle shadow

---

### Loop 4: Comparison — Side-by-Side Analysis

**Trigger:** User wants to compare weather between 2–3 saved cities.

1. User clicks "Compare" in the dashboard header — compare mode activates
2. Cards gain a selection affordance; user clicks to toggle cities into the comparison
3. Selection counter shows how many cities are selected ("2 selected")
4. When 2+ cities are selected, the app auto-transitions to the comparison view
5. Comparison shows columns: each city with current conditions, alerts, hourly, and 7-day forecast
6. User can remove a city from comparison via ✕ on the column header
7. Dropping below 2 cities automatically returns to dashboard

**Feedback signals:**
- Selected cards gain a purple border
- Selection counter shown in prose ("Select at least 2 cities to compare (N selected)")
- Auto-transition to comparison view (no explicit confirm required)

---

### Loop 5: Settings — Preference Configuration

**Trigger:** User wants to change how data is displayed.

1. User clicks "⚙️ Settings" toggle — panel expands inline
2. User selects temperature unit (°C / °F), wind unit (km/h / mph), or time format (24h / 12h)
3. Changes apply immediately to all rendered weather data
4. User can also set a default city from their favorites list
5. Default city is loaded automatically on the next app start

**Feedback signals:**
- Active setting button is highlighted (purple)
- Panel collapses with same toggle button ("Hide Settings" / "Settings")

---

## State Machine: View Transitions

```
[weather]  ←→  [dashboard]  ←→  [compare]
    ↑               ↑
    └── Search / Favorites chip selects a city
```

- `weather → dashboard`: click "Dashboard" tab (only visible when favorites exist)
- `dashboard → weather`: click "Weather" tab or select a city card
- `dashboard → compare`: select 2+ cities in compare mode (auto-transition)
- `compare → dashboard`: click "← Back to Dashboard" or drop below 2 cities
- Any view → weather: select a city from search or favorites bar

---

## Error and Edge States

| Situation | Behavior |
|-----------|----------|
| No city selected | Empty state: "Search for a city to see the weather forecast" |
| Weather fetch fails | Inline error: "Error: [message]" |
| No favorites | Favorites bar hidden; Dashboard tab hidden |
| Dashboard location fails | Per-card "Unable to load" message |
| Comparison location fails | Per-column "Unable to load weather" message |
| Removing default city | Default city preference cleared silently |
