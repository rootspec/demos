# Technical Conventions

## Framework Stack

- **Framework:** Astro 5.18.1
- **Styling:** Tailwind CSS 3.4.19
- **Interactive Components:** React 19.2.4
- **Build Tool:** Vite 6.4.1
- **Testing:** Cypress 14.5.4 with RootSpec integration
- **TypeScript:** 5.9.3

## Project Structure

- **Pages:** `/src/pages/` - Astro pages
- **Layouts:** `/src/layouts/` - Reusable page layouts
- **Components:** `/src/components/` - Astro and React components
- **Styles:** Tailwind utility classes, minimal custom CSS

## Development

- **Dev Server:** `npm run dev` (port 3000)
- **Build:** `npm run build`
- **Test:** `npm run test` (Cypress integration tests)
- **Preview:** `npm run preview`

## Testing Patterns

- **Test Selectors:** `data-test` attributes for all interactive elements
- **Test Location:** `cypress/e2e/mvp.cy.ts`
- **Test Format:** YAML embedded as string literals using loadAndRun pattern
- **Core DSL Steps:** visit, click, fill, shouldExist, shouldContain

## Configuration Files

- **Astro Config:** `astro.config.mjs` with Tailwind and React integrations
- **Tailwind Config:** `tailwind.config.js` with content paths
- **Cypress Config:** `cypress.config.ts` with RootSpec reporter
- **TypeScript Config:** Auto-generated by Astro
