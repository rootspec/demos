# L2: Truths

## Design Philosophy

**Demonstration over documentation** — The site IS the argument for RootSpec, not just text about it.

**Structured transparency** — Show the methodology working in a way that builds confidence rather than confusion.

**Progressive disclosure** — Start with simple concepts, build to complex interactions, maintain clarity throughout.

## Trade-offs

**We choose transparency over polish** — Better to show honest results from minimal input than to hide the rough edges behind manual refinement.

**We choose interactivity over static explanation** — Users should experience the methodology, not just read about it.

**We choose developer authenticity over corporate marketing** — Technical depth and honest limitations rather than buzzwords and overselling.

**We choose client-side simplicity over server complexity** — All interactions run in the browser without external dependencies.

**We choose framework demonstration over feature completeness** — This site proves RootSpec works, not that it produces perfect products without iteration.

## Success Criteria

**Primary Success:** Developers visiting this site can accurately explain what RootSpec does and evaluate whether it fits their workflow within 10 minutes.

**Adoption Metric:** Visitors click through to the GitHub repo and getting started guide at higher rates than industry baseline for developer tools.

**Comprehension Proof:** Interactive features are used (hierarchy explorer clicked, wizard completed) indicating engagement with core concepts.

**Confidence Builder:** Meta banner and spec transparency increase rather than decrease visitor trust in the methodology.

## Constraints

**Performance Budget:** Site loads and becomes interactive within 3 seconds on mobile connections.

**Accessibility Floor:** All interactive features work with keyboard navigation and screen readers.

**Maintenance Reality:** Site content and version numbers can be updated by editing markdown and config files, no developer intervention required.

**Deployment Simplicity:** Static site generation for maximum reliability and minimal hosting complexity.
