# Truths (L2)

*References: [[01.PHILOSOPHY]]*

## Design Philosophy

**Demonstration Over Documentation**: We choose live interactive examples over text explanations. The hierarchy explorer and spec wizard teach better than paragraphs about methodology.

**Constraints Breed Creativity**: We choose client-side-only architecture over server complexity. Structured templates and progressive disclosure create rich interactions without external dependencies.

**Honest Minimalism**: We choose transparent rough edges over polished deception. A site built from ~100 lines of product description with minimal iteration honestly shows what RootSpec produces.

## Strategic Trade-offs

**Simplicity over Completeness**: We choose essential RootSpec concepts over comprehensive documentation. Better to communicate the core methodology clearly than overwhelm with edge cases.

**Interactivity over Information**: We choose hands-on experience over detailed explanations. Visitors learn by doing (wizard, explorer) rather than reading extensive marketing copy.

**Transparency over Polish**: We choose visible development process over hidden complexity. Meta-banner, version badges, and GitHub links show exactly how this was built.

**Speed over Sophistication**: We choose fast, functional interactions over elaborate animations. Smooth transitions enhance usability without distracting from content.

**Foundation over Features**: We choose accessibility and responsive design over advanced interactivity. Every feature works for every user before adding complexity.

## Success Criteria

**Immediate Comprehension**: 90% of developer visitors understand what RootSpec is within 2 minutes of landing on the hero section.

**Engagement Conversion**: 40% of visitors interact with either the hierarchy explorer or spec wizard before leaving.

**Repository Interest**: 15% of engaged visitors click through to the GitHub repository or view the spec/seed files.

**Methodology Clarity**: Visitors can explain the 5-level hierarchy and reference flow after exploring the interactive sections.

**Demo Validation**: The site itself demonstrates every claim made about RootSpec—no feature exists without tracing to a design pillar.

## Constraints

**Technical**: 
- Astro static site generation
- Client-side JavaScript for interactivity
- No external APIs or server dependencies
- Mobile-first responsive design
- Keyboard accessibility compliance

**Content**: 
- All copy traces to specific SEED.md requirements
- No marketing buzzwords or corporate language
- Technical accuracy about RootSpec methodology
- Honest disclosure about demo limitations

**Performance**: 
- Sub-3-second initial load time
- Smooth 60fps animations and transitions
- Progressive enhancement for slower connections
- Functional core experience without JavaScript

**Scope**: 
- Focus on core RootSpec methodology (5 levels + reference flow)
- Essential interactive demonstrations only
- Single-page application experience
- Version v7.0.7 framework compatibility
