# Truths

## Design Philosophy

**Validation over generation** — The real value lies in extracting and validating human intent, not in code generation. AI has knowledge but not wisdom; humans supply philosophy.

**Demonstration over explanation** — Interactive experiences teach concepts more effectively than descriptive text. Abstract methodologies become concrete through hands-on exploration.

**Transparency over perfection** — Honest communication about capabilities builds more trust than polished marketing claims.

## Strategic Trade-offs

**We choose simplicity over completeness**
*OVER* comprehensive feature sets that obscure the core value proposition. Every section serves the central narrative of philosophy-driven development.

**We choose client-side interactions over dynamic server features**
*OVER* complex backend integrations that introduce failure points. Reliability matters more than sophistication for demonstrating concepts.

**We choose authentic developer voice over corporate messaging**
*OVER* polished marketing copy that feels disconnected from the technical reality. Our audience values directness.

**We choose mobile-first responsive design over desktop-optimized experiences**
*OVER* layouts that work poorly on touch devices. Technical professionals increasingly work on varied devices.

**We choose keyboard accessibility over mouse-only interactions**
*OVER* visually striking but navigationally limited interfaces. Inclusion drives adoption in developer communities.

**We choose framework version visibility over hiding implementation details**
*OVER* abstract presentations that don't acknowledge the underlying technology. This site is itself a RootSpec demo.

## Success Criteria

**Immediate comprehension** — 80% of visitors understand what RootSpec is within 2 minutes, measured by time spent on hierarchy explorer and progression to other sections.

**Practical engagement** — 60% of visitors who stay longer than 3 minutes interact with either the hierarchy explorer or spec wizard, indicating active learning rather than passive reading.

**Authentic transparency reception** — Meta banner generates positive sentiment in developer discussions, demonstrating that honest communication about limitations builds rather than undermines credibility.

**Conversion to exploration** — 25% of engaged visitors click through to view the actual spec files or seed document, showing genuine interest in the methodology.

**Accessibility compliance** — 100% keyboard navigability and screen reader compatibility across all interactive features, with zero critical accessibility violations.

## Constraints

**Performance budget:** Page loads in under 2 seconds on 3G connections. Interactive features respond in under 100ms. No external API dependencies that could introduce latency.

**Content constraint:** All copy fits the "thoughtful developer" voice — technical but accessible, confident but not preachy, no buzzwords or corporate speak.

**Maintenance constraint:** Single-constant updates for framework version display. No hardcoded version strings scattered throughout the codebase.

**Implementation constraint:** Must work with Astro's static site generation model. No features that require server-side rendering or complex state management.

**Browser constraint:** Full functionality in last 2 major versions of Chrome, Firefox, Safari, and Edge. Graceful degradation for older browsers.
