# Level 3: Interaction Architecture

References: [L1: Foundational Philosophy], [L2: Stable Truths]

## Interaction Loop Architecture

| Scale     | Scope                        | Duration     | Key Patterns                            |
|-----------|------------------------------|--------------|-----------------------------------------|
| Immediate | Single element interaction   | Seconds      | Click, expand, toggle, fill, animate    |
| Session   | Page visit → engaged reading | Minutes      | Scroll, explore, wizard completion      |
| Extended  | Return visits, sharing       | Days/Weeks   | Bookmarking, link sharing, referrals    |
| Lifetime  | Adoption of methodology      | Ongoing      | Framework integration into daily work   |

## Core Interaction Patterns

### Pattern 1: Linear Narrative Scroll
The page flows top-to-bottom through a coherent argument: problem → methodology → interactive proof → call to action. Each section builds on the last. Users who read linearly arrive at the CTA having already experienced the methodology firsthand.

**Trigger:** Page load
**Feedback:** Smooth section transitions, progress anchored by section headers

### Pattern 2: Meta-Transparency Banner
A persistent banner near the top of the page explains that the site was itself built using RootSpec. This banner includes links to the spec and seed files in the public repository.

**Trigger:** Always visible
**Feedback:** Static — text and links. Should not animate or close. Honest and direct.

### Pattern 3: Hierarchy Explorer (Interactive)
Users can click any of the five spec levels to expand it and view example content. Visual connections (arrows) show which levels each can reference — arrows flow upward only, visualizing the dependency inversion rule. Hovering a level highlights its allowed references.

**Trigger:** User clicks a level card
**Feedback:**
- Selected level expands to show example content
- Arrow overlays highlight allowed reference paths
- Other levels may visually dim or reduce
- Keyboard navigation (arrow keys / tab / enter) supported

**Failure modes:**
- On narrow screens, collapsible accordion replaces visual diagram
- If no level is selected, all levels show their title and a one-line description

### Pattern 4: Spec Wizard (Interactive, Multi-step)
A three-step mini-wizard where the user enters a product idea and walks through the first three levels of a RootSpec spec:

1. **Step 1 — Mission:** User enters a one-line product idea; selects or writes a mission statement using provided templates
2. **Step 2 — Design Pillars:** User selects [minimum pillars] to [maximum pillars] emotional descriptors from a curated list, or writes their own
3. **Step 3 — Key Interaction:** User describes one core user interaction in free text

**Output:** A skeleton spec fragment showing how their input maps to L1-L3, formatted to look like an actual RootSpec file.

**Trigger:** User clicks "Try It" or scrolls to wizard section
**Feedback:**
- Step indicator shows current position and progress
- Each step validates before allowing advance (mission cannot be blank, at least [minimum pillars] pillar selected)
- Output animates into view on wizard completion
- Wizard can be reset to try again

**Failure modes:**
- User submits blank mission: inline validation message, no advance
- User tries to skip pillars: prompt to select at least [minimum pillars]
- On mobile: wizard stacks vertically, touch targets are finger-sized

### Pattern 5: Before/After Comparison
A side-by-side (or toggle) comparison showing the contrast between a vague traditional spec and a structured RootSpec spec for the same product. Real content in both panels — not lorem ipsum.

**Trigger:** User scrolls to section or clicks toggle
**Feedback:**
- On desktop: side-by-side columns with a labeled divider
- On mobile: tab toggle (Before / After) — one panel visible at a time
- Visual differentiation between panels (e.g., color, typography, density)

### Pattern 6: Theme Toggle
Users can switch between dark and light mode manually. System preference is detected on first load.

**Trigger:** Click theme toggle button (header or floating)
**Feedback:** Immediate visual transition, preference persisted in local storage

## Multi-Scale User Journey

### First Visit (Session)
1. Banner catches attention → user notes this site is itself a RootSpec demo
2. Hero communicates the core value proposition and current version
3. Problem section resonates with a pain the user has felt
4. How It Works walkthrough maps the four skills to a mental model
5. Hierarchy Explorer lets the user interact with the five-level structure
6. Spec Wizard gives the user a first-person taste of the methodology
7. Before/After Comparison makes the contrast viscerally clear
8. CTA directs user to the GitHub repo for next steps

### Return Visit or Direct Link (Session)
User arrives at a specific section via a shared link or bookmark. Each section is self-contained enough to be meaningful without context from prior sections.

## Edge Cases and Failure Modes

- **JavaScript disabled:** Core content (hero, problem, how-it-works) remains readable as static HTML. Interactive sections show a "requires JavaScript" fallback message.
- **Slow connection:** Content loads progressively; interactive components hydrate after core content is visible
- **Mobile / narrow viewport:** Hierarchy Explorer collapses to accordion, Before/After uses tabs, Wizard stacks vertically
- **Keyboard-only navigation:** All interactive elements reachable and operable via keyboard; focus states are visible
- **Missing `.rootspec.json`:** Version badge falls back to a static placeholder text rather than breaking the build
