# L3: Interactions

## Core User Journey

**First 30 seconds:** Visitor lands on hero, reads tagline, immediately sees meta banner explaining this site is itself a RootSpec demo. Banner links provide instant credibility check.

**First 2 minutes:** Scroll through problem statement and "how it works" section. Visceral recognition of specification pain points builds motivation to learn more.

**First 5 minutes:** Engage with hierarchy explorer. Click through L1-L5, see visual connections, understand the "upward reference only" rule through interaction rather than explanation.

**First 10 minutes:** Complete the "Spec Your Idea" wizard. Input their own product concept, walk through methodology steps, see structured output that maps their thinking to L1-L3 framework.

## Interaction Loops

### Discovery Loop (One-time)
**Trigger:** Organic search, social media, developer recommendation  
**Flow:** Hero → Problem → How It Works → Interactive exploration → GitHub/CTA  
**Feedback:** Each section builds on previous understanding; interactive elements provide "aha moments"  
**Success:** Visitor understands RootSpec value and knows how to get started

### Exploration Loop (Repeatable)  
**Trigger:** Return visits, deeper investigation  
**Flow:** Direct to interactive features → Compare before/after → Explore spec files → Reference docs  
**Feedback:** Increased familiarity with methodology details  
**Success:** Visitor becomes confident enough to try RootSpec on their own project

### Reference Loop (Utility)
**Trigger:** Using RootSpec, need methodology reminder  
**Flow:** Direct link to specific sections → Quick visual reference → Return to work  
**Feedback:** Information found quickly without distraction  
**Success:** Visitor gets answer and leaves satisfied

## Primary Interaction Patterns

### Hierarchy Explorer
**Input:** Click on any of the 5 levels (L1-L5)  
**Process:** Level expands to show example content, visual arrows highlight allowed reference directions  
**Feedback:** Hover states show connections, clicked level stays expanded until new selection  
**Edge Cases:** All levels can be expanded simultaneously; mobile touch handles multiple selections

### Spec Wizard  
**Input:** Three-step form: product idea → mission templates → design pillars → key interaction  
**Process:** Each step builds on previous, shows methodology in practice with user's own content  
**Feedback:** Real-time preview of spec structure, clear progress indication  
**Edge Cases:** Can restart at any step, handles empty inputs gracefully, no server dependency

### Before/After Comparison
**Input:** Slider or toggle control  
**Process:** Side-by-side view transitions between chaotic vs structured specification approaches  
**Feedback:** Smooth animation, real content in both panels shows concrete difference  
**Edge Cases:** Mobile stacks vertically, comparison content stays aligned

### Theme Toggle
**Input:** Manual theme button or system preference detection  
**Process:** Instant theme switch with smooth transition, preference persistence  
**Feedback:** Visual confirmation of theme change, respects system dark mode  
**Edge Cases:** Handles system theme changes while site is open

## Failure Modes

**JavaScript Disabled:** Core content remains readable, interactive features show fallback explanations rather than broken UI.

**Slow Connections:** Progressive enhancement ensures basic functionality loads first, enhancement layers add when ready.

**Small Screens:** All interactive features adapt to touch interfaces, no desktop-only interactions.

**Accessibility Tools:** Keyboard navigation works for all interactive elements, screen readers can use hierarchy explorer and wizard.

**External Link Failures:** GitHub links failing don't break site functionality, alternate contact methods available.

## Error States

**Wizard Input Validation:** Clear inline feedback for required fields, no server errors to handle.

**Missing Content:** If spec files referenced in meta banner aren't accessible, fallback to repository link with explanation.

**Theme Persistence Issues:** Graceful degradation to system preference, manual override always works.

## Feedback Systems

**Immediate:** Hover states, click responses, form validation, smooth transitions  
**Progressive:** Wizard step completion, exploration progress through hierarchy  
**Contextual:** Error messages inline, success states for completed actions  
**Persistent:** Theme preference, wizard completion state during session
