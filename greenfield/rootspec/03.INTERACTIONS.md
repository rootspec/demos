# Level 3: Interaction Architecture

## Interaction Loop Architecture

| Scale     | Scope                        | Duration   | Key Patterns                               |
|-----------|------------------------------|------------|---------------------------------------------|
| Immediate | Single click or scroll       | Seconds    | Expand/collapse, toggle, form input         |
| Session   | Page exploration              | Minutes    | Section discovery, wizard completion        |
| Extended  | Return visit or referral     | Days/Weeks | Sharing, return to complete wizard          |
| Lifetime  | Adoption of RootSpec          | Ongoing    | Not tracked by this site (no accounts)      |

## Core Interaction Patterns

### Pattern 1: Progressive Disclosure (Hierarchy Explorer)
The Hierarchy Explorer presents five collapsed levels. Users click to expand and see example content. Visual arrows indicate allowed reference directions (upward only). Hovering a level highlights its allowed references.

**Trigger:** User curiosity about the five-level structure
**Feedback:** Level expands with content; reference arrows animate
**Reset:** Clicking an open level collapses it
**Keyboard:** Arrow keys navigate levels; Enter/Space toggle expansion

*References: Clarity Over Completeness (L1), Demo IS the explanation (L2)*

### Pattern 2: Guided Wizard (Spec Your Idea)
A three-step wizard where the user enters a product idea and walks through:

1. **Step 1:** Mission selection — text input or template selection
2. **Step 2:** Design pillars — checkbox selection from [N] suggestions, choose 3-5
3. **Step 3:** Key interaction — free-text description of one core user interaction

**Output:** A rendered skeleton spec showing L1 (mission + pillars) and L3 (interaction) mapped to RootSpec levels. No L2, L4, L5 generated — the output is intentionally incomplete to show methodology, not produce a finished spec.

**Trigger:** User clicks "Try It" or equivalent CTA
**Feedback:** Progress indicator shows current step; output updates in real time as user types
**Completion:** "Your spec skeleton" appears with a callout to run `/rs-init`
**Keyboard:** Tab navigates fields; Enter advances steps

*References: Approachability Without Sacrifice (L1), Natural language is the spec (L2)*

### Pattern 3: Comparison Toggle (Before/After)
A side-by-side or toggled view showing:

- **Without RootSpec:** Vague requirements, ambiguous stories, untraceable decisions
- **With RootSpec:** Structured hierarchy, testable stories, every feature traces to a design pillar

Real content in both panels (not lorem ipsum). Toggle switches between views on mobile; side-by-side on desktop.

**Trigger:** Scrolling into the comparison section
**Feedback:** Panels render with distinct visual treatments; toggle animates transition
**Keyboard:** Toggle button is focusable; Spacebar activates

*References: Validation as First Principle (L1), Human wisdom + AI execution (L2)*

### Pattern 4: Theme Toggle
Persistent dark/light mode toggle. System preference detected on first load. Manual override stored in localStorage. No flash of wrong theme on page load.

**Trigger:** User clicks theme toggle
**Feedback:** Entire page transitions smoothly between themes
**Persistence:** Preference survives page reload
**Keyboard:** Toggle button focusable, activatable by Enter/Space

*References: Approachability Without Sacrifice (L1)*

## Passive Interaction Patterns

### Scroll-Based Section Entry
Sections animate into view as user scrolls. Smooth, non-distracting fade or slide. No autoplay video or audio. Works on reduced-motion preference.

### Meta Banner (Persistent)
Always visible above or below hero. Contains links to SEED.md and spec files on GitHub (absolute URLs). Cannot be dismissed. Its persistence is intentional — every visitor must encounter it.

## Navigation Flow

```
Landing → Hero → Meta Banner → Problem → How It Works
  → Hierarchy Explorer → Spec Wizard → Before/After → CTA → Footer
```

Single-page scroll. No routing. All sections are in view order.

## Failure Modes and Edge Cases

### Wizard abandonment
User starts wizard but does not complete all steps. Output section shows a partial skeleton with a prompt to complete remaining steps. State persists for the session (no persistence beyond page reload).

### Mobile: Hierarchy Explorer
On small screens, reference arrows are hidden. Level labels truncate to [N] characters. Expansion shows full content in a scrollable panel below the level list.

### Reduced motion
All animations respect `prefers-reduced-motion: reduce`. Transitions that involve movement become instant or fade-only.

### No JavaScript
The page renders all static content without JavaScript. Interactive components degrade to static examples. A visible notice explains that interactive features require JavaScript.
