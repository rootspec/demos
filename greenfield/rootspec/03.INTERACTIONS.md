# Level 3: Interaction Architecture — HOW (Conceptual)

> References: 01.PHILOSOPHY.md, 02.TRUTHS.md

## Core Interaction Model

The marketing site is a single-page scrolling experience. Visitors arrive, orient, explore, and convert. No login, no state persistence across sessions, no forms that submit to a server. All interactive elements run client-side.

The primary journey: land → understand → experience the methodology → decide to try it.

## Interaction Loop Architecture

| Scale     | Scope                        | Duration    | Key Behavior                                    |
|-----------|------------------------------|-------------|--------------------------------------------------|
| Immediate | Single click/keystroke       | <1s         | Visual feedback (expand, highlight, toggle)     |
| Session   | One visit                    | 3–10 min    | Scroll through content, try interactive features |
| Extended  | Return visits                | Days/weeks  | Deeper reading, sharing, comparing notes        |
| Lifetime  | Ongoing engagement with spec | Months+     | Referencing back to the methodology             |

## Primary Interaction Flows

### Flow 1: Linear Discovery (most visitors)
1. Visitor arrives at page, sees hero section and meta banner
2. Scrolls through problem statement, solution overview
3. Encounters Hierarchy Explorer — clicks levels to understand the methodology
4. Tries the Spec Wizard — enters their idea, walks through three steps
5. Sees the Before/After comparison
6. Reaches CTA — clicks to GitHub

**Triggers:** Scroll, click-to-expand, wizard navigation
**Feedback:** Animated section transitions, interactive highlights, spec output preview

### Flow 2: Direct Access (developer shared a link)
1. Visitor arrives mid-page or with an anchor
2. May jump directly to an interactive section
3. Explores that section, then scrolls up or down for context

**Triggers:** URL anchor, direct section navigation
**Feedback:** Section is immediately usable without requiring prior sections

### Flow 3: Meta Banner Inspection (methodology-curious visitor)
1. Visitor notices meta banner: "This site was built with RootSpec"
2. Clicks "View the spec →" or "View the seed →"
3. Opens GitHub in new tab to read spec files
4. Returns to site with new context

**Triggers:** Meta banner links
**Feedback:** External navigation (new tab), no disruption to main page flow

## Interactive Feature Specifications

### Hierarchy Explorer

**Purpose:** Make the five-level hierarchy tangible — not just a list, but an explorable structure.

**Behavior:**
- Five levels displayed as expandable cards or a visual tree
- Each level shows: name, icon, purpose (one sentence), example content
- Clicking a level expands it to show full detail
- Visual arrows or connections show reference flow (arrows point upward only — L5 points to L4, L4 to L3, etc.)
- Hovering a level highlights the levels it can reference (dimming others)
- Keyboard navigable: Tab between levels, Enter/Space to expand, Escape to collapse

**States:**
- Default: All levels collapsed, showing names and one-sentence summaries
- Expanded: One level open with full content; others remain collapsed
- Hover: Reference connections highlighted

**Failure modes:**
- If JavaScript fails: All levels display in expanded static state (progressive enhancement)
- On touch devices: Tap to expand/collapse; hover effects replace with tap-and-hold or are omitted

### Spec Wizard ("Spec Your Idea")

**Purpose:** Let the visitor experience the methodology by applying it to their own idea in 90 seconds.

**Behavior:**
- Step 1: Text input — "What's your one-line product idea?" Free-text entry.
- Step 2: Mission selection — 3–4 mission templates to choose from, or free-text. User selects one and can edit.
- Step 3: Design pillars — Grid of [N] suggested emotional qualities. User selects 3–5.
- Step 4 (output): Skeleton spec showing how their inputs map to L1–L3 format. Copyable.

**States:**
- Step indicator shows current step (1/3, 2/3, 3/3)
- Back/Next navigation between steps
- Output panel appears after step 3 completes
- "Start over" resets to step 1

**Keyboard:** Tab through choices, Enter to select, arrow keys in grids, Enter on "Next" advances

**Failure modes:**
- Empty input: "Next" button disabled until minimum content provided
- Step validation: Require at least 1 mission selection and 3 pillar selections before output

### Before/After Comparison

**Purpose:** Contrast the experience of unstructured vs. RootSpec-structured requirements.

**Behavior:**
- Two panels side-by-side (desktop) or stacked (mobile)
- Toggle or slider to switch focus between panels
- Real content in both panels — not placeholder text
- Without RootSpec panel: vague requirements doc, ambiguous user stories, no traceability
- With RootSpec panel: structured hierarchy, traceable stories, design pillar references

**States:**
- Default: Both panels visible equally
- Toggle: Active panel receives visual emphasis
- Mobile: Panels stack vertically with clear labels

### Theme Toggle

**Purpose:** Honor system preferences; allow manual override.

**Behavior:**
- On load: Read `prefers-color-scheme` media query and apply matching theme
- Toggle button in header: switches between light and dark
- Theme persists for duration of session (or localStorage if available)
- Smooth transition between themes (no flash)

**Failure modes:**
- If localStorage unavailable: Theme resets on page reload (acceptable)
- If media query unsupported: Default to dark theme

## Navigation Patterns

### Section Navigation
- The page is a single scroll; sections are anchored
- Header (if present) provides jump links to major sections
- Active section highlighted in navigation as user scrolls

### External Links
- GitHub links (spec, seed, CTA) open in new tab
- No external links open in the same window without clear affordance

## Edge Cases and Failure Modes

| Scenario | Behavior |
|----------|----------|
| JavaScript disabled | Static content fully readable; interactive sections degrade gracefully to static displays |
| Slow network | Page loads content progressively; no spinner-only states blocking core content |
| Mobile screen | All interactive features usable with touch; no horizontal overflow |
| Keyboard-only navigation | All interactive elements reachable and operable via keyboard |
| Very long user input in wizard | Input truncated or scrollable within the field; output panel handles it gracefully |

## Verification Criteria

- A visitor can understand what RootSpec is without interacting with any JavaScript
- A visitor can experience the hierarchy interactively without reading any documentation
- A visitor can complete the Spec Wizard in under 2 minutes
- The meta banner is visible without scrolling on any viewport width
- All links to GitHub resolve to correct files
