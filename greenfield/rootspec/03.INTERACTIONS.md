# Level 3: Interaction Architecture

**Version:** 7.3.2
**Status:** Draft

---

## Interaction Loop Architecture

| Scale     | Scope                        | Duration      | Key Patterns                         |
|-----------|------------------------------|---------------|--------------------------------------|
| Immediate | Single section or element    | Seconds       | Hover, click, expand, toggle         |
| Session   | Single page visit            | Minutes       | Scroll journey, wizard completion    |
| Extended  | Return visits                | Days/Weeks    | Methodology adoption, sharing        |
| Lifetime  | RootSpec practitioner        | Ongoing       | Version updates, community           |

## Core Interaction Patterns

### Pattern 1: Linear Scroll Journey

The primary interaction is a single-page scroll from top to bottom. Each section builds on the previous: meta-banner → hero → problem → methodology → interactive features → CTA. The user forms a mental model by moving through the page.

**Trigger:** Page load / initial visit
**Feedback:** Smooth section transitions with animation on entry; no jarring jumps

### Pattern 2: Theme Toggle

The user can switch between dark and light modes. System preference is detected on load; the toggle persists across reloads.

**Trigger:** Toggle button click in header
**Feedback:** Immediate visual transition; the toggle reflects current state

### Pattern 3: Hierarchy Explorer Expand/Collapse

The user clicks (or keyboards) a level in the five-level hierarchy visualization. The selected level expands to show example content. Arrows connecting levels highlight to show which levels the selected level can reference. Hovering or focusing a level shows its allowed references.

**Trigger:** Click or keyboard (Enter/Space) on a hierarchy level card
**Feedback:** Smooth expand animation; arrows animate to indicate reference direction; non-referenced levels dim slightly

**Touch support:** Tap behaves identically to click

### Pattern 4: Spec Your Idea Wizard

A three-step guided wizard. The user enters a product idea and is guided through selecting a mission, choosing design pillars, and describing a key interaction. The output is a skeleton spec fragment showing how their input maps to L1-L3.

**Steps:**
1. Enter product idea (text input)
2. Select or write mission (template choices or freeform)
3. Select design pillars (multi-select from suggestions, minimum [small count], maximum [moderate count])
4. Describe one key interaction (text input)
5. Output panel shows the spec skeleton

**Trigger:** User initiates wizard; "Next" advances steps; "Back" returns; "Generate Spec" completes
**Feedback:** Step indicator shows progress; transitions between steps are animated; output panel appears with the spec result

**Touch support:** All controls are touch-friendly; tap targets meet minimum size requirements

**Keyboard navigation:** Tab/Shift-Tab moves through form elements; Enter submits; Escape clears

### Pattern 5: Before/After Comparison Toggle

The user toggles between a "without spec" and "with RootSpec" view of the same product scenario. Content is real — not lorem ipsum.

**Trigger:** Toggle button or slider interaction
**Feedback:** Smooth transition between panels; active state is clearly indicated

### Pattern 6: Meta-Banner Link Navigation

The meta-banner contains two external links: one to the SEED.md and one to the spec files in the GitHub repository. These open in new tabs.

**Trigger:** Click on "View the spec →" or "View the seed →"
**Feedback:** Standard browser link behavior; new tab opens

## Failure Modes and Edge Cases

### Wizard incomplete submission
If the user attempts to advance without completing a required field, the field is highlighted and a brief inline message appears. The spec output is only shown when all steps are complete.

### Theme preference unavailable
If `prefers-color-scheme` is not available, the site defaults to dark mode.

### No JavaScript
The page must render meaningfully without JavaScript. Interactive components (wizard, hierarchy explorer) degrade gracefully — showing static content rather than broken UI.

### Small screen layout
On mobile, the hierarchy explorer stacks levels vertically instead of in a grid. The wizard remains single-column. The before/after comparison shows one panel at a time with a toggle.

## Verification Criteria

- The five-level hierarchy can be fully explored (all levels expandable) without a mouse
- The wizard produces a valid spec skeleton for any non-empty product idea
- Theme toggle works on first click without page reload
- Meta-banner links open correct GitHub URLs
- Comparison toggle shows distinct, real content in both states
- All interactive elements meet WCAG AA touch target and contrast requirements
