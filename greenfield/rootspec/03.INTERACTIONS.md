# L3: Interaction Architecture

## Core User Journey

A developer lands on the page (typically via a link from a conference talk, blog post, or colleague). The first 60 seconds:

1. **Meta banner** — immediately visible, explains: "This site was generated from a ~100-line product description using the RootSpec pipeline." Links to the seed and spec files. Visitor understands the context before reading anything else.
2. **Hero** — tagline and one-sentence explanation. Version badge visible. Immediate clarity on what RootSpec is.
3. **The Problem** — visitor sees their pain described accurately. Spec drift, AI hallucination, dead Google Docs. They lean in.
4. **How It Works** — four-step visual walkthrough of the skills. Not a wall of text.

From here, the visitor explores interactives in any order:
- **Hierarchy Explorer** → clicks a level → sees content expand → understands the dependency chain
- **Spec Wizard** → enters a product idea → walks three steps → sees a skeleton spec → realizes it's approachable
- **Before/After** → toggles or slides → sees the contrast → feels the value

Ends at the **Open Source CTA** — GitHub link, getting started, community.

## Interaction Patterns

### Meta Banner
- Persistent — always visible, not dismissable (this is a demo site, context matters throughout)
- Inline links to SEED.md and spec files on GitHub
- No interaction required — informational only

### Theme Toggle
- **Trigger:** Click the toggle in the header
- **Behavior:** Switches between light and dark mode; persists via localStorage
- **Default:** Follows OS system preference on first visit
- **Feedback:** Immediate visual switch, smooth transition

### Hierarchy Explorer
- **Trigger:** Click any level (L1–L5) in the visualization
- **Behavior:** Clicked level expands to show example content; arrows animate to show allowed upward references; previously expanded level collapses
- **Hover behavior:** Highlights which levels the hovered level can reference
- **Keyboard:** Arrow keys navigate between levels; Enter/Space expand/collapse; Escape collapses all
- **Mobile:** Tap to expand; swipe not required; all content accessible via vertical scroll

### Spec Your Idea Wizard
- **Entry:** User types a one-line product idea into a text input
- **Step 1:** Select or write a mission statement (3 template options + free-write)
- **Step 2:** Pick 3–5 design pillars from a suggested list (chips/tags, multi-select)
- **Step 3:** Describe one key interaction in a text area
- **Output:** A formatted skeleton spec panel showing L1–L3 populated with their inputs
- **Navigation:** Back/Next buttons; no form submission; entirely client-side
- **Keyboard:** Tab through fields; Enter advances from Step 3 to output; Escape resets wizard
- **Mobile:** Single-column layout; large tap targets; output scrolls naturally below wizard

### Before/After Comparison
- **Trigger:** Toggle switch OR drag a slider (two modes for accessibility)
- **Behavior:** Reveals "Without RootSpec" and "With RootSpec" panels
- **Content:** Real structured content in both panels — not lorem ipsum
- **Keyboard:** Toggle via Space on the switch; slider via arrow keys
- **Mobile:** Toggle mode default (slider is secondary); both panels readable at mobile width

### Section Scroll
- Smooth scroll to sections via anchor links in navigation
- No full-page routing — single page
- Scroll position preserved on theme toggle

## Failure Modes and Edge Cases

- **Wizard with empty input:** Next button disabled until minimum content entered; clear inline hint
- **Wizard reset:** A "Start over" link resets all three steps without page reload
- **Hierarchy explorer on very small screens:** Falls back to an accordion-style stacked list if the circular/graph visualization doesn't fit; all content still accessible
- **Theme toggle with no localStorage:** Falls back to system preference silently; no error shown
- **Long product idea in wizard:** Text area enforces a character limit (200 chars); counter shown near limit
- **Before/After on mobile:** Toggle is the primary mode; slider degrades gracefully or is hidden

## Feedback Patterns

- Interactive elements respond within 1 frame of interaction (no loading states required — everything is local)
- Hover states on all clickable elements
- Focus rings visible for keyboard navigation
- Wizard step progress shown via a simple step indicator (Step 1 of 3)
- Expanded hierarchy level is visually distinct (border, background, or highlight — per theme)
