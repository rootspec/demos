# L3 — INTERACTIONS
*HOW users and product interact*

## Primary User Journey

**First moments:** Visitor arrives → sees clear tagline and one-sentence explanation → notices meta banner explaining this site was generated from minimal input → understands this is both a product demo and the product itself

**Initial exploration:** Scrolls through problem statement → recognizes their own pain points with spec drift and unclear requirements → begins to understand RootSpec as a solution

**Deep engagement:** Explores hierarchy visualization → clicks through levels to see example content → grasps the core concept of philosophy-first design → tries the "Spec Your Idea" wizard with their own concept

## Core Interaction Loops

### Discovery Loop (One-time)
**Trigger:** Landing on site  
**Pattern:** Read hero → explore problem section → understand "How It Works"  
**Feedback:** Progressive revelation builds understanding  
**Completion:** Clear mental model of what RootSpec is

### Exploration Loop (Repeated)
**Trigger:** Curiosity about methodology details  
**Pattern:** Click hierarchy level → read examples → see reference connections → understand constraints  
**Feedback:** Visual highlights show allowed references, illegal references grayed out  
**Completion:** Concrete understanding of hierarchy rules

### Experimentation Loop (Repeated)  
**Trigger:** "Could this work for my project?"  
**Pattern:** Enter product idea in wizard → select mission template → choose design pillars → describe key interaction  
**Feedback:** Real-time skeleton spec generation shows methodology application  
**Completion:** Personalized example they can relate to their work

### Comparison Loop (Investigation)
**Trigger:** Skepticism about benefits  
**Pattern:** Toggle between "without spec" and "with spec" examples → see concrete differences  
**Feedback:** Side-by-side slider reveals transformation  
**Completion:** Clear evidence of methodology value

### Action Loop (Conversion)
**Trigger:** Convinced to try RootSpec  
**Pattern:** Click GitHub link → follow getting started instructions → run first commands  
**Feedback:** Links to actual runnable code and documentation  
**Completion:** Active trial of methodology

## Interaction Patterns

### Theme Toggle
**When:** User prefers different visual mode or system changes  
**How:** Single click/tap toggles between light/dark themes with smooth transition  
**Feedback:** Icon changes, colors transition smoothly, preference persists

### Hierarchy Explorer  
**When:** User wants to understand RootSpec structure  
**How:** Click level to expand content, hover to highlight allowed references  
**Feedback:** Visual arrows show reference flow, content slides in/out  
**Edge Cases:** Keyboard navigation with arrow keys and enter

### Spec Wizard
**When:** User wants to try methodology on their idea  
**How:** Enter idea → step through mission/pillars/interaction → see generated skeleton  
**Feedback:** Real-time validation, next step unlocked only when current complete  
**Edge Cases:** Invalid input shows helpful guidance

### Before/After Comparison
**When:** User questions methodology value  
**How:** Drag slider or click toggle to reveal transformation  
**Feedback:** Content morphs from vague requirements to structured spec  
**Edge Cases:** Touch-friendly drag targets on mobile

## Error States and Edge Cases

**Hierarchy Explorer - No JavaScript:** Graceful degradation to static content with all levels visible  
**Spec Wizard - Invalid Input:** Inline validation with clear error messages and suggestions  
**Theme Toggle - System Override:** Respects user's explicit choice over system preference  
**Mobile Navigation:** Touch targets meet accessibility guidelines ([minimum touch target size])  
**Slow Connections:** Progressive enhancement with meaningful content before JavaScript loads
