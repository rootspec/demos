# Philosophy (L1)

## Mission

Create a marketing website that demonstrates RootSpec's power by being built entirely with RootSpec. Show developers how philosophy-driven specification produces coherent, testable products where every feature traces to a design principle.

## Design Pillars

**Trust through Transparency**: Visitors immediately understand this site is a live demo built from minimal human guidance. The meta-banner honestly shows the spec, seed, and generation process—rough edges included.

**Show, Don't Tell**: Interactive hierarchy explorer and spec wizard let visitors experience the methodology firsthand rather than reading marketing copy. Every claim about RootSpec is demonstrated on the site itself.

**Confident Simplicity**: Technical precision without corporate buzzwords. Like a thoughtful developer explaining a methodology they believe in. Clean visual hierarchy, smooth interactions, no unnecessary complexity.

**Accessible Depth**: Essential concepts accessible to any developer, with deeper technical details available for those who want them. Mobile-friendly interactions, keyboard navigation, progressive disclosure.

**Living Validation**: The site itself proves RootSpec works by existing. Every section connects back to a spec level. Version badges and GitHub links make the development process transparent.

## Problem

Existing specification approaches fail developers and teams:

- **Spec Drift**: Google Docs specifications become outdated the moment code is written
- **Philosophy Gap**: Implementation details drive product decisions instead of user experience principles
- **Unreliable AI Output**: AI tools generate code without validation against human intent
- **Circular Dependencies**: Requirements, design, and code influence each other in unpredictable ways
- **Untraceability**: Features exist without clear connection to user value or business goals

## North Star

A developer visits this site, understands RootSpec in 5 minutes, tries the spec wizard with their own product idea, sees how their input maps to the 5-level hierarchy, and leaves confident they can apply this methodology to eliminate spec drift and improve AI-assisted development. Six months later, they're using RootSpec to keep their team aligned on complex products where philosophy guides implementation, not vice versa.

## Inviolable Principles

**Philosophy First**: Every feature, interaction, and design decision must trace upward to a design pillar. Never add features for their own sake.

**Honest Demonstration**: This site is proof-of-concept, not production polish. Rough edges from minimal human guidance are features, not bugs—they prove the methodology works with sparse input.

**Client-Side Only**: All interactive features (wizard, hierarchy explorer, comparison tool) run client-side with structured templates. No external APIs, no AI calls, no server dependencies.

**Accessibility Foundation**: Every interactive element works on mobile and via keyboard. Visual hierarchy accommodates all users.

**Version Transparency**: The RootSpec version used to build this site is prominently displayed and easily updateable.
