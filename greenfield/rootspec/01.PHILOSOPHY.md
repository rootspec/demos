# L1 — PHILOSOPHY
*WHY and WHAT EXPERIENCE*

## Mission

RootSpec implements purpose-aligned declarative specification where philosophy guides implementation, never vice versa. This marketing site demonstrates the methodology by being itself a living example — built from a ~100-line product description with no manual code or design mockups.

## Problem Statement

Software teams struggle with spec drift, the philosophy-implementation gap, unreliable AI output without validation, and Google Docs specs nobody reads. When code and specs can be generated trivially, the real value is validation and proof. AI has knowledge but not wisdom — humans supply philosophy, AI executes implementation, and the spec bridges the gap.

## Target Audience

**Primary:** Developers and technical leads who have been burned by scope drift or unclear requirements, are using AI tools (Copilot, Claude, etc.) and want more reliable output, lead teams where "the spec" is a Google Doc nobody reads, or build complex products that need to stay coherent over time.

## Design Pillars

- **Transparent Demonstration** — Every rough edge reveals minimal human guidance, not carelessness
- **Approachable Methodology** — Complex concepts made tangible through interaction, not academic theory  
- **Confident Authority** — Technical but accessible, like a thoughtful developer explaining a methodology they believe in
- **Immediate Clarity** — Visual and interactive proof of concepts, not verbose explanations
- **Living Validation** — The site itself proves the methodology works by existing

## Inviolable Principles

- **Philosophy First:** The hierarchy flows downward only — L1 Philosophy → L2 Truths → L3 Interactions → L4 Systems → L5 Implementation
- **No Circular Dependencies:** Each level can only reference higher levels, never lower ones
- **Executable Specification:** Not a code generator but a validation gate that extracts and validates human intent
- **Framework Agnostic:** Works with or without AI, any tooling, any team structure
- **Natural Language:** Specification in human language, not config/YAML-first approaches

## North Star

Six months from now, visitors immediately understand that RootSpec solves the philosophy-implementation gap. They can experience the methodology firsthand through interactive features, see concrete before/after examples, and have a clear path to try it themselves. The site demonstrates its own creation story transparently — this is what you get from sparse guidance through the RootSpec pipeline, and that's exactly the point.
